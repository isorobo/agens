<objective>
Author the agens read-only recommend/refuse spine as one Claude Code Skill. After this plan, a user invokes agens by /agens or by describing a new agent project, answers the four fixed questions in one call, and receives either a citation-grounded pattern recommendation (resolvable vault path plus a quoted trigger/trade-off passage, gated behind a Read+Grep check) or a plain refusal that cites nothing.

Purpose: The recommend/refuse spine is the phase's core value — every recommendation is grounded in a verified vault citation, and the skill refuses rather than dressing a near-miss as a fit.
Output: `.claude/skills/agens/SKILL.md` and `.claude/skills/agens/references/trigger-tests.md`.
</objective>

These three CONTEXT.md gray areas ("Not discussed this session") are resolved here against REQUIREMENTS.md and the index entry format, per the CONTEXT.md instruction to resolve or raise. None materially changes the single-file deliverable, so they are resolved in-plan (not raised to the user). A fourth item below (grep-only recall) resolves RESEARCH.md Open Question 4 on the same basis:

- **Match & refusal threshold (RECOMMEND-05/06):** A match requires a single bold entry in `agent-patterns-index.md` whose Trigger the four answers satisfy — the goal bucket maps to a pattern family AND the workflow, sensitivity, and latency answers do not contradict that entry's Trigger. If no bold entry's Trigger fits, or the answer is the tool's free-text "Other", agens refuses.
- **Recommendation output format (RECOMMEND-04):** Quote the matched entry's Trigger and Trade-off sentences verbatim. Surface exactly ONE pattern by default (REQUIREMENTS.md Out of Scope forbids citing multiple notes "to look thorough").
- **Goal-bucket wording (Claude's Discretion):** Use the four D-03 labels verbatim; add NO catch-all "something else" bucket. `AskUserQuestion` always appends a free-text "Other" option that cannot be suppressed, so a manual catch-all is redundant and would waste an option slot; the tool's "Other" routes to refusal.
- **Grep-only recall (RESEARCH.md Open Question 4, STATE.md concern):** No recall risk from a goal-bucket wording that does not lexically overlap a Trigger. Pattern selection is LLM-semantic goal-family mapping (Task 3), not a literal grep of the goal-bucket label against the note's Trigger text. Grep runs only AFTER a candidate pattern is chosen, as a post-hoc check that the bold **Name** literal exists in the note (Task 3 gate). A lexical mismatch between a goal bucket and a Trigger therefore cannot cause a false refusal on a supported project; only a genuinely absent bold entry refuses. This recall behaviour is empirically validated across all four goal buckets by the Plan 01-03 checkpoint (one matching example per bucket). No synonym expansion or search index is built (out of scope).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user → questionnaire | Untrusted answers, including the tool's free-text "Other", cross into agens |
| vault note → context | `agent-patterns-index.md` content is read into the model context and may carry adversarial or stale text |
| skill body → filesystem | Read/Grep/Glob calls reach the granted vault directory |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing (of grounding) | Recommendation output | mitigate | No recommendation without the Read (path resolves) + Grep (bold `**Name**` present) gate; both must pass (RECOMMEND-05, Task 3) |
| T-01-02 | Tampering | Questionnaire answer handling | mitigate | Treat free-text "Other" on any dimension as a non-match and route to refusal; never fabricate a match from free text (Pitfall 2, Task 3) |
| T-01-03 | Elevation of privilege (prompt injection) | Vault note read into context | mitigate | Quote the matched Trigger/Trade-off as data, not as instructions; the read-only grant (Read Grep Glob AskUserQuestion) prevents any note-directed write or shell action |
| T-01-04 | Tampering | Vault path lookup | mitigate | Reference the note by the fixed vault-relative path `30_Concepts/agent-patterns-index.md`; no user-controlled path segment reaches Read/Grep, blocking path traversal |
| T-01-05 | Elevation of privilege | Tool grant | mitigate | allowed-tools limited to Read Grep Glob AskUserQuestion; no Write, Edit, or Bash in Phase 1 |
| T-01-SC | Tampering | npm/pip/cargo installs | accept | No package installs this phase — agens is markdown driven by built-in tools (RESEARCH.md Package Legitimacy Audit: N/A) |
</threat_model>
